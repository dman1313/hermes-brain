# Shared Memory Vault

The agent-memory vault (`~/agent-memory/`) is the fleet-wide shared memory. All agents (Hermes, Claude Code, Codex, Goose, Kimi, Kiro) read and write here via Git.

**Repo:** dman1313/agent-memory-coding
**VPS path:** `~/agent-memory/`
**Mac path:** `/Users/dwayne-primeau/Library/Mobile Documents/com~apple~CloudDocs/Agent Memory/Coding`

## Session Protocol

Every session:

1. `cd ~/agent-memory && git pull --rebase`
2. Read `NOW.md` — current fleet state (who's active, what's blocked, recent milestones)
3. Check `AGENT-CHANNEL.md` — pending inter-agent messages
4. Do work
5. Update `NOW.md` via `build-context.sh` or manually
6. Append entries to `ACTIVITY.md` (below `<!-- ENTRIES BELOW THIS LINE -->` marker)
7. Record significant decisions in `DECISIONS.md`
8. `git add -A && git commit -m "hermes: <summary>" && git push`

## Key Files

| File | Purpose | Edit Style |
|------|---------|------------|
| `NOW.md` | Current state dashboard | Auto-generated by `build-context.sh`, or manual |
| `ACTIVITY.md` | Append-only activity log | Prepend entries below marker line |
| `DECISIONS.md` | Architectural decisions | Append new entries, never delete |
| `AGENT-CHANNEL.md` | Inter-agent messages | To[@Agent] format, mark DONE when handled |
| `Agents/<name>.md` | Per-agent profiles | Update when capabilities change |
| `CONTEXT.md` | Consolidated context | Auto-generated, don't edit directly |
| `MEMORY.md` | Index with wikilinks | Update when adding new files |

## ACTIVITY.md Entry Format

```
YYYY-MM-DDTHH:MM:SSZ | agent-name | event-type | project-slug | detail text
```

Event types: `session-start`, `session-end`, `decision`, `blocker`, `blocker-resolve`, `milestone`, `handoff`, `note`

Agent names: `hermes`, `claude-code`, `codex`, `goose`, `kimi`, `kiro`, `antigravity`

**Critical:** Entries must be below `<!-- ENTRIES BELOW THIS LINE -->` marker for `build-context.sh` to parse them.

## DECISIONS.md Entry Format

```md
### YYYY-MM-DD — [Title]
- **Context:** What situation prompted this decision
- **Decision:** What was decided
- **Agents:** Who was involved
- **Project:** Which project this affects
- **Status:** active | superseded | reversed
```

## Pitfalls

- **ACTIVITY.md marker missing:** `build-context.sh` looks for `<!-- ENTRIES BELOW THIS LINE -->` — without it, NOW.md generates with 0 entries. Always include this marker.
- **build-context.sh Mac path:** Originally hardcoded to Mac iCloud path. Now auto-detects VPS (`/home/ubuntu/agent-memory`) vs Mac. If it fails, check the path detection at the top of the script.
- **Push before pull:** Always `git pull --rebase` before editing to avoid merge conflicts with Mac agents.
- **Don't edit CONTEXT.md directly:** It's auto-generated. Edit the source files (NOW.md, ACTIVITY.md, etc.) and regenerate.

## Cross-Agent Skill Sharing

When you need to teach another agent a skill (e.g., mac-hermes), use the vault as the transfer mechanism:

1. **Copy SKILL.md files** to a staging directory in the vault:
   ```bash
   mkdir -p ~/agent-memory/skills-for-mac/<skill-name>
   cp ~/.hermes/skills/<category>/<skill-name>/SKILL.md ~/agent-memory/skills-for-mac/<skill-name>/
   # Also copy references/ if they exist
   cp ~/.hermes/skills/<category>/<skill-name>/references/*.md ~/agent-memory/skills-for-mac/<skill-name>/references/ 2>/dev/null
   ```

2. **Write an install script** (`skills-for-mac/install.sh`) that copies them into the target agent's `~/.hermes/skills/`:
   ```bash
   #!/bin/bash
   HERMES_SKILLS="$HOME/.hermes/skills"
   VAULT_DIR="$HOME/agent-memory/skills-for-mac"
   mkdir -p "$HERMES_SKILLS/<category>/<skill-name>"
   cp "$VAULT_DIR/<skill-name>/SKILL.md" "$HERMES_SKILLS/<category>/<skill-name>/SKILL.md"
   ```

3. **Commit + push** the vault changes.

4. **Leave an AGENT-CHANNEL message** with:
   - What skills are available
   - Install instructions (`git pull` + `bash install.sh`)
   - Any prerequisites (e.g., CLI tools to install, auth steps)
   - Mark as `PENDING` — receiving agent marks `DONE` when installed

5. **Adapt SKILL.md paths** if the target agent runs on a different OS (e.g., Mac paths differ from VPS paths).

- second-brain: unprocessed raw items, unpushed commits

Silent when healthy. Add to HEARTBEAT.md for periodic checks.
